How to use field
1

Create field.json

Define your sprints, owners, dependencies, and deadlines in one config file.

$ npx @grainulator/field init # Creates field.json with sprint definitions
2

Plan the work

Field analyzes dependencies and shows the critical path. See what is blocked and what can run in parallel.

$ field plan
3

Check status

One command shows all sprints, their progress, and any cross-sprint conflicts.

$ field status
Real example

4 parallel sprints with dependencies

Your team is researching auth scaling, data migration, API design, and performance. Some depend on others.

$ field status 4 sprints tracked. 2 active, 1 done, 1 blocked. Sprint Status Claims Owner Deadline Depends On ------------------------------------------------------------------------ auth-scaling * active 23 alice 2026-03-20 - data-migration * active 17 bob 2026-03-22 - api-design x done 41 carol - - perf-baseline ! blocked 8 - 2026-03-25 auth-scaling $ field plan Dependency graph: api-design (done) auth-scaling -----> perf-baseline (blocked, waiting) data-migration Critical path: auth-scaling -> perf-baseline Parallel work: data-migration can proceed independently Conflict: auth-scaling/r005 contradicts data-migration/r012 (both claim different max connection limits)
Key features

Research at organizational scale

When one sprint is not enough. Coordinate, detect conflicts, and see the big picture.

Dependency Graphs

Sprint B needs Sprint A's results. Field tracks it and shows the critical path so nothing gets stuck.

Conflict Detection

Two sprints reach opposing conclusions? Field flags it before it becomes a problem.

Team Assignment

See who is running what, who is overloaded, and which sprints need owners.

Unified Dashboard

One self-contained HTML file. Every sprint, every claim, every deadline. Share it with leadership.
